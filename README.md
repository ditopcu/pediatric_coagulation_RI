# Pediatric Coagulation Reference Intervals

Repository: <https://github.com/ditopcu/pediatric_coagulation_RI>

Code and intermediate results supporting the manuscript on age- and sex-specific
reference intervals (RIs) for PT, aPTT, and fibrinogen in children aged 1 to 18
years, derived from retrospective hospital data on Roche Cobas t511/t711
analysers (Turkish tertiary center).

The work combines three estimation strategies:

- **Indirect refineR** (CRAN package `refineR`)
- **Direct CLSI EP28-A3c** estimates from an a posteriori healthy subset
- **Continuous GAMLSS** age-dependent intervals using the Ammer et al. pipeline

A side analysis covers infants aged 30 to 365 days (Supplementary Figure S2 and
Table). Department-based exclusion sensitivity is reported in Supplementary
Analysis 4.

## Reproducibility model

Raw, identifiable patient data are **not redistributed**. Two reproducibility
layers are provided:

1. **Results-level reproducibility** -- all intermediate result objects are
   committed: per-test refineR fits (`data/processed/*.RDS`), the GAMLSS model
   files (`cont_out/*.RData`), the publication tables (`data/processed/tables/`),
   and the rendered figures (`figures/`). Sourcing `src/pub.R`,
   `src/analysis.R`, or `src/tables.R` regenerates the published figures and
   tables from these objects without touching raw data.
2. **Pipeline-level reproducibility** -- the analysis code is fully open and
   parameterised; if you have a dataset with the schema described below, the
   pipeline can be re-run end-to-end via `src/run_all.R` followed by the GAMLSS
   `RUNNER_Pipeline_*.R` scripts.

## Required environment

- R 4.4.x or 4.5.x on Windows / macOS / Linux
- CRAN packages: `tidyverse`, `readxl`, `refineR`, `gamlss` (+ `gamlss.dist`,
  `gamlss.data`, `nlme`), `patchwork`, `ggrepel`, `cowplot`, `janitor`,
  `moments`, `openxlsx`, `officer`, `flextable`
- The `src/pipeline/` directory contains the Ammer GAMLSS pipeline (unmodified)
  and is required for the continuous RI estimation.

Pinned package versions are recorded in `renv.lock` (regenerated by
`renv::snapshot()`) or in `docs/sessionInfo.txt` for environments where `renv`
is not used.

## Quick reproduction (no raw data)

After cloning, from the project root:

```r
setwd("path/to/this/repo")
source("src/common.R")     # loads shared setup
source("src/pub.R")        # regenerates the 6 manuscript figures from RDS/RData
source("src/tables.R")     # regenerates the 18 CSV tables
source("src/analysis.R")   # regenerates Analyses 1-6 (FP rates, transferability, etc.)
```

Each script writes to `data/processed/`, `data/processed/tables/`, or
`figures/`. Existing files are overwritten in place.

## End-to-end reproduction (with raw data)

If you obtain a properly licensed copy of the de-identified source dataset:

1. Place the Excel files under `data/coa_results/` with the filenames and schema
   described below.
2. Run `Rscript src/coa\ hesap\ 3.R` to regenerate the refineR RDS objects.
3. Run `Rscript "src/RUNNER_Pipeline pt.R"` (and the aPTT / Fib variants) to
   regenerate the GAMLSS `cont_out/*.RData` objects.
4. Run `Rscript src/run_all.R` to regenerate all figures, tables, and analyses
   from the freshly produced intermediate objects.

## Data schema

The pipeline expects three Excel files representing the main 1-18 y dataset
plus three legacy files containing infants. The minimum schema is:

**Main 1-18 y dataset** -- `data/coa_results/tce 2026 1-18 {pt,aptt,fib}.xlsx`

| column        | type      | notes                                              |
|---------------|-----------|----------------------------------------------------|
| `patient_id`  | character | de-identified patient identifier                   |
| `sample_id`   | character | de-identified sample identifier                    |
| `department`  | character | requesting clinic / unit (e.g., "COCUK ACIL")      |
| `sex`         | character | "K" (Female) or "E" (Male)                          |
| `age_year`    | numeric   | age at sampling, decimal years (1.0 to <18.0)      |
| `age_month`   | numeric   | age at sampling, integer months                    |
| `age_group`   | character | optional integer-year bin                          |
| `test_name`   | character | "PT", "aPTT", or "Fibrinogen"                      |
| `result_num`  | numeric   | measurement value (s or g/L)                       |

**Infant supplementary dataset** -- `data/coa_results/{PT-2,aPTT,Fib-2} 2025.10.17.xlsx`

| column        | type      | notes                                              |
|---------------|-----------|----------------------------------------------------|
| `sample_id`   | character | de-identified                                      |
| `sex`         | character | "K" (Female) or "E" (Male)                          |
| `age_year`    | numeric   | decimal years; rows with 30/365 <= age_year < 1.0 are the infant subset |
| `age_group`   | character | optional bin                                       |
| `result_num`  | numeric   | measurement value                                  |

Datasets must already exclude any direct identifiers and inpatient ICU samples,
as documented in the manuscript Methods.

### Synthetic smoke-test dataset

A tiny synthetic dataset is bundled under `data/synthetic/` and is generated by
`src/_make_synthetic_data.R`. It mirrors the column schema above but the
numerical values are drawn from broad clinically plausible distributions and
**do not reproduce the published numbers**. Use it only to verify that the
pipeline runs end-to-end without errors.

## Run order summary

| Layer | Script(s) | Purpose | Input | Output |
|---|---|---|---|---|
| 0 | `common.R` | shared setup | (none) | global objects in R session |
| 1 | `coa hesap 3.R` + `RUNNER_Pipeline_{pt,aptt,fib}.R` | refineR + GAMLSS fits | raw Excel | `data/processed/*.RDS`, `cont_out/*.RData` |
| 2 | `analysis.R`, `analysis_raw_comparison.R`, `analysis4_sensitivity.R` | derived results | raw Excel + RDS/RData | `data/processed/analysis*.csv`, comparison Excel |
| 3 | `tables.R` | publication tables | RDS/RData + analysis CSVs | `data/processed/tables/*.csv` |
| 4 | `export_main_xlsx.R`, `export_supplementary_xlsx.R` | Excel bundles | CSV tables | `PUB_main_tables.xlsx`, `PUB_supplementary_tables.xlsx` |
| 5 | `pub.R` | publication figures | RDS/RData + analysis CSVs | `figures/{TIFF_600DPI,PNG_300DPI}/figure_*.{tiff,png}` |
| 6 | `infant_*.R` and `build_supplementary_methods.R` | supplement artefacts | RDS/RData + CSVs | infant figure, table, Word, Supplementary Methods |

Steps 2-6 are wrapped by `src/run_all.R` for routine regeneration.

## Caveats

- Within-run analytical performance values produced by
  `src/tables.R` (Supplementary Table S2 block) are illustrative
  placeholders; the authoritative values are reported directly in the published
  manuscript's Supplementary Table S2.
- The 3SD-per-test outlier filter is used **only** for the Figure 1
  distribution display; all reference-interval and false-positive-rate
  analyses operate on the unfiltered RAW data.
- For aPTT, age partitioning at 12 years is applied prior to refineR and
  GAMLSS estimation, based on a Harris-Boyd z-test (see Supplementary Methods).
